# 薬液調合ノート(オフライン対応PWA)

農薬タンクミックスの薬量・水量計算と防除記録アプリ。
一度開けば圏外の圃場でも動作します。データは各端末内に保存されます。

## ファイル構成

```
index.html                    … 本体ページ
app.js                        … アプリ本体(コンパイル済み)
react.production.min.js       … React本体(ローカル同梱・CDN不要)
react-dom.production.min.js   … ReactDOM(同上)
manifest.json                 … PWAマニフェスト(ホーム画面追加用)
sw.js                         … Service Worker(オフラインキャッシュ)
icon-192.png / icon-512.png   … アプリアイコン
apple-touch-icon.png          … iPhone用アイコン
```

## GitHub Pagesでの公開手順

### 1. リポジトリを作成
GitHubで新規リポジトリを作成(例: `tankmix-note`)。Public にする
(Privateの場合、Pagesは有料プランが必要)。

### 2. ファイルをアップロード
このフォルダの中身(8ファイル)をすべてリポジトリの**ルート直下**に置く。

ブラウザから: リポジトリ画面 → 「Add file」→「Upload files」→ 8ファイルをドラッグ&ドロップ → Commit

またはコマンドで(WSLから):
```bash
cd site
git init
git add .
git commit -m "薬液調合ノート 初版"
git branch -M main
git remote add origin https://github.com/<ユーザー名>/tankmix-note.git
git push -u origin main
```

### 3. Pagesを有効化
リポジトリの Settings → Pages →
- Source: 「Deploy from a branch」
- Branch: 「main」/「/ (root)」 → Save

1〜2分後、`https://<ユーザー名>.github.io/tankmix-note/` で公開されます。

### 4. スマホのホーム画面に追加
公開URLをスマホで開いて:
- **iPhone**: Safariで開く → 共有ボタン → 「ホーム画面に追加」
- **Android**: Chromeで開く → メニュー(⋮) → 「ホーム画面に追加」または「アプリをインストール」

以後はアイコンから起動でき、**圏外でも動作**します。

## 更新のしかた

1. 変更したファイル(通常は app.js や index.html)をリポジトリに上書きプッシュ
2. **sw.js の1行目 `CACHE_VERSION` の数字を上げる**(例: tankmix-v1 → tankmix-v2)
   ※これを忘れると各端末に古いキャッシュが残り続けます
3. 各端末では、オンライン状態で一度開き直すと新版に切り替わります

## Googleスプレッドシート連携(記録の自動集約)

各端末の記録を1枚のスプレッドシートに自動で集める機能です。
圏外で保存した記録は端末に貯まり、電波が戻ると自動送信されます。

### 設置手順(最初に1回だけ)

1. **Googleスプレッドシートを新規作成**(名前は自由。例:「防除記録」)
2. メニュー **拡張機能 → Apps Script** を開く
3. 最初から入っているコードを全部消して、同梱の **Code.gs** の内容を貼り付けて保存
4. 右上 **デプロイ → 新しいデプロイ** → 歯車アイコンで種類 **ウェブアプリ** を選択
   - 次のユーザーとして実行: **自分**
   - アクセスできるユーザー: **全員** ←ここが重要
5. **デプロイ** を押す(初回は「アクセスを承認」→自分のアカウントを選択
   →「詳細」→「(プロジェクト名)に移動」→ 許可、と進む)
6. 表示された **ウェブアプリのURL**(`https://script.google.com/macros/s/…/exec`)をコピー

### 各スマホでの設定(1台につき1回)

1. アプリの **記録タブ →「設定」** を開く
2. **送信先URL** にコピーしたURLを貼り付け
3. **記録者名** に名前を入力(シートの「記録者」列に入ります)
4. **接続テスト** を押して「✅接続OK」が出れば完了

以後、「記録に保存」するたびに自動でシートに1行ずつ追加されます。
記録の右上に「✓送信済」/「未送信」バッジが表示されます。

### スクリプトを更新した場合

Code.gsを修正したときは、デプロイ → **デプロイを管理** → 鉛筆アイコン →
バージョン「**新バージョン**」→ デプロイ、で反映します(URLは変わりません)。

## 注意

- 記録・プリセットは**各端末のブラウザ内**(localStorage)に保存されます。
  端末間の同期はありません。集約したい場合は各自CSV出力してください。
- ブラウザのデータ消去(履歴削除で「サイトデータ」を含めた場合)を行うと
  記録も消えます。大事な記録は定期的にCSV出力を推奨します。
- 混用可否・混和順序は必ず各薬剤のラベル・メーカー指示を優先してください。
